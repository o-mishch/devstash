# web/ — Vite SPA

The `web/` workspace is the Vite + TanStack Start SPA. This file covers only what is **settled**: its gates and its generated files. Architectural idioms (routing, guards, server-state ownership) are still in flux and live in `context/current-feature.md` until they stop moving — do not treat that doc's frontend notes as standing rules.

## Testing — there are none, by decision

**`web/` ships zero tests. Do not add a test runner, a test config, or a test file under `web/`.**

There is no Vitest, no Jest, no component-test setup, and no `*.test.*` / `*.spec.*` file anywhere in the workspace — this is a decision, not an oversight or a gap to fill. An audit must not report "missing tests" as a finding here.

Frontend correctness is proven by the gates below plus browser verification, not by unit tests.

## Gates

Run these from `web/`. All must pass before the work is done:

```bash
npm run typecheck   # tsc --noEmit
npm run lint        # oxlint .
npm run build       # typecheck + scripts typecheck + OG image + vite build + finalize-dist
```

- `npm run build` is the real gate, not a formality — it is the **only** place the production SPA render path, the `/` prerender, and the CSP hash pipeline are exercised. Dev runs an SSR dev server; prod builds SPA mode. Green in dev proves nothing about prod.
- Runtime behavior is verified **in a browser**, not by a test suite.
- Formatting is `oxfmt` (`npm run format`), not Prettier.

## Generated files — never hand-edit

Regenerate these; a hand edit is a finding. This table is mirrored by the `generated:` frontmatter key above — the machine-readable copy the cleanup skill reads to keep these out of a lens audit. **Keep the two in step**; `globcheck.ts` fails when they disagree.

| Path | Generated by | Reproducible offline |
| --- | --- | --- |
| `web/src/client/**` | `npm run openapi:gen` (Hey API, from `web/openapi.json`) | yes |
| `web/src/routeTree.gen.ts` | TanStack Router plugin | yes |
| `web/src/styles/themes.generated.css` | `npm run themes:gen` (from `presets-raw.json`) | yes |
| `web/src/lib/theme-presets.generated.ts` | `npm run themes:gen` (from `presets-raw.json`) | yes |
| `web/openapi.json` | the Go backend's `openapi` subcommand | no — needs the Go toolchain |
| `web/firebase.json` | `scripts/finalize-dist.ts` at build time — edit `firebase.template.json` instead | no — build-time only |
| `web/src/lib/themes/presets-raw.json` | `npm run themes:fetch` — a vendored tweakcn snapshot, the *input* to `themes:gen`, not its output | no — network fetch |
| `web/src/components/ui/**` | the shadcn CLI (Base UI flavor; see `components.json`) — vendored from the registry | no — network fetch |
| `web/public/og-image.png` | `scripts/gen-og-image.tsx` at build time (satori) — gitignored, like `firebase.json` | no — build-time only |
| `web/package-lock.json` | `npm install` | no — network fetch |

"Reproducible offline" is what an audit can actually prove: those rows are verified by regenerating into a temp dir and diffing. The rest are declared generated so they are not hand-audited, but a hand edit in them cannot be detected without the network or a build — an audit reports them `unverifiable` rather than clean.

`web/.oxlintrc.json` already excludes `src/client/**`, `src/routeTree.gen.ts`, and `scripts/**` from lint. `scripts/**` is type-checked through `tsconfig.scripts.json` (wired into `npm run build`) rather than linted — that is deliberate, not a hole.

## TypeScript

`web/` pins TypeScript to `^5.9.3` and keeps `exactOptionalPropertyTypes` **off**. Both are required for generated-client compatibility and are the one sanctioned exception to the latest-version policy. Do not "fix" either.

Stack-agnostic TypeScript rules apply here via `.agents/rules/typescript-standards.md`, React rules via `react.md`, and Tailwind rules via `tailwind.md` — all three already glob `web/**/*`. `legacy-coding-standards.md` does **not** apply here: there is no `src/stores/`, no Server Actions, and no `server-only` boundary in this workspace.
