# P1 — Architecture

Check whether logic lives in the correct layer and whether a simpler structure would remove debt.

Layer boundaries are defined by the governing rule for each changed path, not by a list here.
Audit each changed file against its rule. There is deliberately no path list in this file: a copy
drifts from the rule and starts producing both false positives and misses.

Stack-independent signals:

- Access checks, validation, or data access living in UI/presentation code.
- A trust boundary crossed without a mechanism enforcing it — a secret-holding module reachable
  from an untrusted context with nothing that fails the build or the request.
- Flow spread across too many files for the behavior it implements.
- Circular imports or tight coupling between unrelated modules.
