# P2 — KISS and DRY

Work hardest here. Simpler means fewer concepts and less indirection, not always fewer lines. DRY
means one source of truth.

Signals:

- Same guard, conditional, transform, schema shape, error map, token flow, or display rule
  appears in 2+ places. A near-duplicate that differs by one line still counts — name the
  abstraction that unifies them.
- Changed code reimplements something the stack already provides. Before accepting any new
  utility, search the codebase for one that already does it.
  - `src/` — an existing helper, hook, schema, route wrapper, Zustand store, TanStack Query
    pattern, shadcn primitive, Prisma feature, Zod feature, or Next.js primitive.
  - `web/` — an existing `web/src/lib` helper, hook, Zustand store, TanStack Query/Router
    pattern, `web/src/components/ui` primitive, Zod feature, or a generated `web/src/client`
    operation.
  - `backend/` — an existing `internal/` helper, a Huma feature, a sqlc query already in
    `db/queries/**`, or a stdlib built-in.
- `web/` — a TanStack Query key written inline at two or more call sites instead of through a
  shared key factory. The duplicated key shape is the DRY finding, and it is also what makes a
  later `invalidateQueries` miss one copy. Name the factory that unifies them. (Web data-loading
  idioms are not yet standing rules — `web-architecture.md § web/ — Vite SPA` — so file this as an
  audit-lens finding, not a rule deviation.)
- One-use wrapper, one-export file, unused generic, premature abstraction, pass-through prop, or
  deep import chain.
- A flag, option, branch, or parameter that no caller exercises, or that exists only to serve a
  hypothetical future.
- Net LOC growth without equivalent removed code; a 30-line change that a built-in or existing
  helper does in 3.
- State that a derived value would compute, or an effect doing what render-time computation or
  the framework's own data loading already does.
  - `src/` — a client component that could be a server component with less state or fewer props.
  - `web/` — there are no server components in this workspace. Data-loading idioms are not yet
    standing rules (`web-architecture.md § web/ — Vite SPA`); do not file one as a rule deviation.
  - `web/` — a manual `useMemo`, `useCallback`, or `React.memo` added for performance where the
    React Compiler (enabled in `web/`) already memoizes: dead indirection, not an optimization.
    A `useMemo` kept for referential identity a dependency actually needs is not this finding.
- A comment explaining what the code does (rather than why) because the code is harder to read
  than it needs to be.
